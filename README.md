# Quick start - How to to run

Prerequisites

Git: Used for cloning the repository.
Docker: Required to build and run the Docker container.

Clone the project repository from GitHub.

```bash
git clone repository-name-url
cd repository-name
```

Build the Docker image using the Dockerfile in the project directory. This will set up the Vite project and configure Nginx to serve it as a static site.

```bash
docker build -t widgets-vite-app .
```

Run the Docker container from the image built. This command maps port 80 of the container to port 8080 on your host, making the application accessible via http://localhost:8080.

```bash
docker run -p 8080:80 widgets-vite-app
```

Open a web browser and navigate to http://localhost:8080 to view the project served via Nginx.

## Expanding the ESLint configuration

If you are developing a production application, we recommend updating the configuration to enable type aware lint rules:

- Configure the top-level `parserOptions` property like this:

```js
export default tseslint.config({
  languageOptions: {
    // other options...
    parserOptions: {
      project: ['./tsconfig.node.json', './tsconfig.app.json'],
      tsconfigRootDir: import.meta.dirname,
    },
  },
});
```

- Replace `tseslint.configs.recommended` to `tseslint.configs.recommendedTypeChecked` or `tseslint.configs.strictTypeChecked`
- Optionally add `...tseslint.configs.stylisticTypeChecked`
- Install [eslint-plugin-react](https://github.com/jsx-eslint/eslint-plugin-react) and update the config:

```js
// eslint.config.js
import react from 'eslint-plugin-react';

export default tseslint.config({
  // Set the react version
  settings: { react: { version: '18.3' } },
  plugins: {
    // Add the react plugin
    react,
  },
  rules: {
    // other rules...
    // Enable its recommended rules
    ...react.configs.recommended.rules,
    ...react.configs['jsx-runtime'].rules,
  },
});
```
